# jm-doc

Project-documentation router with four modes. Canonical instructions live in the packaged skill at `~/.codex/skills/jm-doc/` — read its `SKILL.md` and follow it, treating everything after `/jm-doc` in this invocation as the mode word and arguments.

Modes: `checkpoint` (session notes into `overview/notes.md`), `spec` (versioned spec interview), `readme` (root README.md), `arc` (root ARCHITECTURE.md). Bare invocation: assess the project and propose a mode (checkpoint bias) — never run one unasked.
